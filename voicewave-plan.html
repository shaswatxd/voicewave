Capabilities

Features

🎙️ Core Voice

  • WebRTC P2P mesh topology — every user connects to every other
  • Socket.IO signaling (offer / answer / ICE candidate exchange)
  • Room create (auto 6-char code) / join by code / leave
  • Password-protected rooms with modal flow
  • Max 20 users per room
  • Auto-delete room when empty

🔊 Audio Processing

  • Mute / Unmute (toggle mic track + broadcast state)
  • Deafen (mute mic + silence all peer output)
  • Per-user volume slider via gain node per peer
  • Master volume + mic gain controls
  • Noise gate (ScriptProcessorNode, configurable threshold 5–40)
  • Noise suppression, echo cancellation, AGC (WebRTC constraints)
  • Speaking detection (RMS-based, threshold 12, debounce 250ms)
  • Audio level visualizer (5-bar meter)
  • ICE / TURN: Google STUN + OpenRelay TURN

💬 Chat

  • Text messaging with timestamps + sender names
  • Emoji quick-insert bar (20 emojis)
  • File sharing (base64, max 5MB — images, pdf, doc, txt, zip)
  • Typing indicators
  • Delete own messages
  • Unread badge counter

🚪 Room Features

  • Room timer (MM:SS elapsed)
  • AFK detection (auto after 5 min inactivity)
  • Raise hand with toast notification
  • Soundboard (10 sounds)
  • Session recording (MediaRecorder, WebM Opus, auto-download)
  • Admin kick users (creator only)
  • Participant list with status icons

🎨 UI

  • Lobby screen (Create/Join tabs, forms, feature pills)
  • Room screen (user grid + side panel)
  • Chat panel (slide-in from right)
  • Settings, Invite, Password, Soundboard modals
  • Toast notifications + connection status indicator
  • 8 avatar gradient colors, custom 4px scrollbar

🔗 Integration

  • URL ?room=CODE auto-fills join form
  • Electron titlebar + tray integration
  • PWA install prompt + service worker caching
  • Responsive design (768px tablet, 480px mobile)
  • WhatsApp share + Web Share API

Deployment

Build & Deploy Pipeline

1

Kill Processes

Kill VoiceWave.exe, electron.exe, node.exe (except self) via taskkill / wmic

2

Generate Icons

SVG → PNG (256 / 16 / 512) + ICO (7 sizes: 16–256) via sharp

3

Sync Versions

Cache-bust timestamps in sw.js, app.html (CSS/JS ?v=), index.html (download link)

4

Clean dist/

rmdir /s /q dist

5

Build Electron

npx electron-builder --win --x64 → NSIS installer + portable exe

6

Git Force Push

git add -A → commit → push --force origin master → Railway auto-deploy

7

GitHub Release

gh release delete (old) + gh release create with installer exe

Rebuild Notes

Known Issues

  • Soundboard is send-only — no audio playback on receive side
  • File sharing uses base64 through Socket.IO (memory-heavy for large files)
  • Recording captures only local audio, not a mixed stream
  • AFK tracker adds event listeners on every room join — memory leak
  • pollPeerSpeaking uses setTimeout recursion that is never fully cleaned up
  • No rate limiting on chat messages
  • No message persistence — all chat is ephemeral
  • No WebRTC reconnection on connection failure
  • publish.js uses Windows-only commands (taskkill, rmdir)
  • Hardcoded TURN credentials (public shared — not production-safe)
  • Version number is duplicated across multiple files
